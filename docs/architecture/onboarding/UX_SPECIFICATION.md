# Especificacao de UX do Onboarding

## Objetivo
Definir a experiencia ideal de uso do Onboarding para administradores, usuarios iniciais, franqueados e consultores de implantacao.

## Escopo
Experiencia, interacoes, feedback, acessibilidade, responsividade e estados de interface do Assistente Guiado de Implantacao.

## Principios de UX
- Mostrar sempre a proxima acao mais importante.
- Explicar por que aquela configuracao importa.
- Reduzir ansiedade de primeiro acesso.
- Guiar sem parecer formulario burocratico.
- Levar o usuario ao primeiro valor o mais rapido possivel.
- Encaminhar a operacao continua para o modulo dono.

## Experiencias obrigatorias
- Wizard inicial com progresso claro.
- Checklist de setup enxuto.
- Resume do ponto em que o usuario parou.
- Retry em falhas.
- Estados de loading com skeleton.
- Estados vazios com acao sugerida.
- Sem permissao com explicacao clara.
- Ajuda contextual por passo.
- Sugestoes de proximas acoes apos setup.

## Decisoes aprovadas
- O franqueado deve ver uma experiencia simplificada orientada a configuracao e primeiro uso.
- O consultor deve apoiar impedimentos de ativacao, nao operar um sistema de projetos dentro do Onboarding.
- A diretoria deve ver indicadores de adocao e first value, nao formulários operacionais.
- Timeline, documentos e tarefas completas pertencem aos modulos donos quando virarem operacao continua.

## Recomendacoes
- Usar linguagem de acompanhamento: "vamos configurar", "proximo passo", "falta pouco".
- Evitar telas longas sem resumo.
- Exibir links diretos para Financeiro, Catalogo, Comunicacao, Documents e outros modulos.
- Nao transformar bloqueios em um BPM visual dentro do Onboarding.
