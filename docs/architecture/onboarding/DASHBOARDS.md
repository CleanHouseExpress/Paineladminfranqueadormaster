# Dashboards do Onboarding

## Objetivo
Definir dashboards, indicadores, cards, graficos, pendencias e alertas de adocao.

## Escopo
Visoes para administrador, diretoria, implantacao, consultor e franqueado. Nao define painel operacional de projetos ou tarefas recorrentes.

## Dashboard administrador
- implementations em setup;
- setups concluidos;
- usuarios proximos do first value;
- passos de configuracao mais pendentes;
- progresso medio de adocao.

## Dashboard diretoria
- curva de adocao;
- tempo medio ate first value;
- unidades com setup pronto;
- gargalos recorrentes de configuracao;
- unidades por regiao.

## Dashboard consultor
- minhas implementations em setup;
- proximas acoes sugeridas;
- impedimentos de ativacao;
- unidades sem first value.

## Dashboard franqueado
- progresso da unidade;
- proximas acoes;
- configuracoes pendentes;
- primeiro valor esperado.

## Decisoes aprovadas
- Dashboard deve priorizar acao, nao apenas metricas.
- RFC-001: indicadores devem ser derivados de implementations.
- RFC-002: indicadores devem medir adocao/setup/first value, nao gestao operacional continua.
- Indicadores devem respeitar escopo de permissao.

## Recomendacoes
- Cards clicaveis devem levar para listas filtradas ou modulos donos.
- Alertas devem explicar causa e proxima acao.
